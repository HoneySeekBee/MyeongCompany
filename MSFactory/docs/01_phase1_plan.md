# Phase 1 — 창고 자율 운반 시스템 상세 기획

## 목표

10대의 바퀴 로봇이 다중 구역 창고에서 태스크 큐를 받아
물건을 자율적으로 픽업 → 이동 → 전달하는 시스템 구축

---

## 창고 레이아웃

- 여러 구역(Zone A, B, C, D...)으로 나뉜 단일 층 창고
- 각 구역마다 스테이션 로봇 1대 고정 배치
- 구역 간 통로에 장애물 없음 (Phase 1 단순화)
- 구체적인 구역 수 및 배치는 ISAAC Sim 씬 설계 단계에서 확정

---

## 시스템 구조

```
외부 태스크 입력
      ↓
┌─────────────────────────────────────────┐
│              태스크 디스패처              │
│  - FIFO 큐 관리                          │
│  - 빈 로봇에 순서대로 태스크 배정         │
│  - 최대 동시 태스크 수: 10 (= 로봇 수)   │
└─────────────────────────────────────────┘
      ↓ 태스크 배정
┌──────────────────────────────────────┐
│          바퀴 로봇 10대 (AMR)         │
│  각 로봇: 물건 1개씩 운반              │
│  자율주행: RL 학습된 정책              │
└──────────────────────────────────────┘
      ↓ 도착
┌──────────────────────────────────────┐
│       스테이션 로봇 (각 구역)          │
│  - 바퀴 로봇에 물건 싣기 (Loading)    │
│  - 바퀴 로봇에서 물건 내리기 (Unload) │
└──────────────────────────────────────┘
```

---

## 태스크 디스패처 규칙

| 규칙 | 내용 |
|------|------|
| 큐 방식 | FIFO (먼저 들어온 순서대로 처리) |
| 태스크 단위 | 물건 1개 이동 (A→B 5개 요청 = 큐에 5개 태스크) |
| 동시 태스크 최대 | 10개 (로봇 대수와 동일) |
| 로봇 배정 | 임무 완료한 로봇이 생기면 큐 앞에서 다음 태스크 픽업 |
| 우선순위 | 없음 (FIFO 순수 적용) |
| 재할당 | 로봇이 현재 임무 완료 후 자연스럽게 새 태스크 배정 |

---

## 바퀴 로봇 동작 흐름

```
[대기] → 태스크 배정됨
  ↓
[출발 구역으로 이동] ← RL 자율주행
  ↓
[스테이션 로봇이 물건 적재] ← Loading
  ↓
[목적 구역으로 이동] ← RL 자율주행
  ↓
[스테이션 로봇이 물건 하역] ← Unloading
  ↓
[대기] → 다음 태스크 배정됨
```

---

## RL 설계 (개별 로봇 자율주행)

### 관측 공간 (Observation)
- 자신의 현재 위치 (x, y)
- 목적지 위치 (x, y)
- 주변 다른 로봇들의 위치 (충돌 회피용)
- 장애물 정보 (Lidar 또는 거리 센서)

### 행동 공간 (Action)
- 선속도 (전진/후진)
- 각속도 (좌/우 회전)
- 연속 행동 공간 (Continuous Action Space)

### 보상 함수 (Reward)
| 보상 | 조건 |
|------|------|
| +대형 | 목적지 도달 |
| +소형 | 목적지에 가까워짐 |
| -대형 | 다른 로봇 또는 벽과 충돌 |
| -소형 | 시간 패널티 (너무 오래 걸리면) |

### 알고리즘
- **PPO** (Proximal Policy Optimization) — 단일 에이전트 기본
- ISAAC Sim 내 병렬 환경으로 대량 학습

## ROS2 / micro-ROS 연동

### 개념

각 로봇에 풀 ROS2는 불가 (ESP32 메모리 부족). 대신 **micro-ROS** 클라이언트를 ESP32에 올리고, 랩탑에서 풀 ROS2를 돌리는 구조.

```
랩탑 (ROS2 Humble 풀스택)
  ├── ISAAC Sim ──ROS2 Bridge──> ROS2 토픽
  ├── Nav2 / 태스크 디스패처
  └── micro-ROS Agent
        ↕ WiFi / UDP
   [XIAO ESP32-S3 × 10대]
   └── micro-ROS 클라이언트
         ├── Subscribe: /robot_N/cmd_vel   ← 이동 명령
         └── Publish:   /robot_N/odom      ← 위치/상태
```

### 시뮬 → 실물 연결 흐름

ISAAC Sim에서 ROS2 브릿지로 내보내는 토픽과 실물 로봇의 micro-ROS 토픽을 동일하게 맞춰두면, 시뮬에서 검증한 코드를 실물에 그대로 사용 가능.

### MCU 변경 사항

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| MCU | XIAO ESP32-C3 (단일코어, 400KB) | XIAO ESP32-S3 (듀얼코어, 512KB) |
| 가격 | 6,000원 | 8,000원 |
| 크기 | 21mm × 17.5mm | 21mm × 17.5mm (동일) |
| 이유 | - | micro-ROS 안정 구동에 메모리 여유 필요 |

### 기술 스택

| 항목 | 기술 |
|------|------|
| 시뮬 ROS2 연동 | ISAAC Sim ROS2 Bridge |
| 중앙 제어 | ROS2 Humble (Ubuntu / WSL2) |
| 로봇 자율주행 스택 | Nav2 (Navigation2) |
| 실물 로봇 통신 | micro-ROS (ESP32-S3) + WiFi/UDP |
| 메시지 브로커 | micro-ROS Agent (랩탑에서 실행) |

## 개발 순서

1. ISAAC Sim 설치 및 기본 씬 구동 확인
2. ISAAC Sim ROS2 Bridge 활성화 확인
3. 창고 레이아웃 USD 씬 제작
4. 바퀴 로봇 에셋 배치 및 물리 설정
5. 스테이션 로봇 배치 (단순 애니메이션으로 시작)
6. OmniIsaac Gym 환경 래핑
7. 단일 로봇 RL 학습 (목적지 이동)
8. 충돌 회피 추가 학습
9. 10대 멀티 에이전트로 확장
10. 태스크 디스패처 연결
11. ROS2 토픽으로 전체 제어 통합
12. 전체 시뮬레이션 통합 테스트

## 성공 기준

- 10대 로봇이 충돌 없이 동시에 운행
- 태스크 큐에서 순서대로 임무 배정 및 완료
- 임무 완료율 95% 이상 (데드락 없음)
- 평균 운반 시간이 최적 경로 대비 1.5배 이내
